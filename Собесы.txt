Тестовое задание для full-stack разработчика

Разработать веб-приложение по заданным критериям:
База данных: PostgreSQL/MongoDB
Бекенд: Node.JS + Express
Фронтенд: Angular, Material

Общие условия и бизнес-логика:
Cущности:
  Пользователь (фио, логин, пароль),
  Товар (наименование),
  Магазин (наименование),
  Документ (магазин, дата, тип документа, товары в документе с указанием кол-ва, кто принял документ)

каждая сущность имеет дату добавления, дату изменения.

Представлен минимальный набор, в случае необходимости можно добавлять другие сущности или поля. Например, товар может обладать Единицей измерения и Штрихкодом.

Задание:
- Создать БД и структуру (модели).
- Вход в веб-приложение по логину/паролю.
- Создать REST API для нужных сущностей.
- Интерфейсы
    * Справочник товаров:
        вывод списка товаров,
        CRUD (REST),
        поиск товара по наименованию.
    * Справочник магазинов:
        вывод списка магазинов,
        CRUD (REST),
        поиск магазина по наименованию.
    * Работа с документами:
        вывод списка документов,
        CRUD (REST),
        фильтр документов: период дат, тип документа, магазин, кто принял,
        поле 'Кто принял' (пользователь) должно автоматически записываться при изменении записи как текущий авторизованный пользователь.

- Заполнить данными (вручную, фикстуры, скрипт, etc.).
- Каждый из интерфейсов содержит индикатор количества найденных записей (номер, в футере списка записей).

Будет плюсом:
- Пользователи обладают правами/входят в группы (например, Кассир/Менеджер/Администратор), добавить ограничение прав на редактирование документов определенного типа только Администратором;
- Веб-интерфейс в стиле Single Page Application (Angular).
- Сформировать приложение на ElectronJS, которое будет оберткой для всего приложения.

Проект должен содержать README.md с описанием шагов, необходимых для запуска.
